# Tonight Data Layer Sprint

## Goal

Tonight's work should strengthen the data asset layer before deeper RAG and Agent development. The current report already proves a 50k-scale pipeline, but three data issues must be fixed before the intelligent agent can feel reliable:

- Full-text evidence is too sparse.
- The year distribution is heavily concentrated in 2026.
- The canonical fields are still metadata-oriented, not yet RAG/Agent-oriented.

## Sprint Tasks

### 1. Year-Balanced Backfill

Target: make 2022, 2023, 2024, and 2025 much closer to 2026 instead of letting 2026 dominate.

Working target:

```text
2022: >= 8k
2023: >= 8k
2024: >= 8k
2025: >= 8k
2026: keep, but avoid adding more 2026-heavy batches
```

Implementation direction:

- Add year-filtered harvesting for OpenAlex, Crossref, PubMed, PMC, arXiv, and DOAJ where supported.
- Save year-balanced batches separately, for example:

```text
data/raw_balanced/openalex/openalex_2022.jsonl
data/raw_balanced/openalex/openalex_2023.jsonl
...
```

- Rebuild analysis assets from both `data/raw` and `data/raw_balanced`.
- Keep the original raw files immutable for reproducibility.

### 2. Full-Text and Evidence Enrichment

Target: increase full-text or long evidence snippets from 4,542 to at least 15,000 records.

Priority sources:

- PMC full text and body excerpts.
- CORE if API key is available.
- Unpaywall/OpenAlex open access URLs as enrichment metadata.
- arXiv PDF/abstract route later if we add PDF parsing.

RAG acceptance:

- Every paper should produce at least one retrievable evidence chunk from title, abstract, keywords, or full text.
- Full-text papers should produce section-aware chunks.
- Every chunk must keep `paper_id`, `chunk_id`, `source`, `year`, `field`, `title`, and `text_source`.

### 3. RAG/Agent-Oriented Schema Upgrade

Current canonical schema is enough for analysis, but not enough for high-quality RAG and recommendation.

Add or derive:

```text
doi
publication_date
venue
url
open_access_url
citation_count
author_ids
institutions
normalized_title
text_source
abstract_length
full_text_length
chunk_count
dedupe_key
```

Why this matters:

- DOI and normalized title improve cross-source deduplication.
- URL/open access fields improve evidence traceability.
- citation_count and venue help recommendation and trend ranking.
- text provenance prevents the agent from pretending an abstract is full text.

## Stop Rule

Do not start advanced RAG/Agent feature work until:

- non-2026 records are materially improved,
- full-text/evidence chunks have a clear growth path,
- canonical schema includes DOI/provenance fields,
- data-layer audit can be rerun from Makefile.
