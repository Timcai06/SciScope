### 项目分层结构（数据 → RAG → 智能体 → 客户端）
```
数据要素/
  data_pipeline/            1. 加载/规范化（loaders·normalize·models）
  src/harvest, src/analysis/ 1. 6 源采集 / 分析资产 -> 数据报告
  src/models/               2. 嵌入·趋势·推荐·图谱·双语·重排
  models/, output/graphs/   2. 模型文件 / 知识图谱 JSON
  backend/app/services/     3. 混合检索(RAG)·GraphRAG·推荐·趋势·图谱
  backend/app/agent/        4. LangGraph StateGraph · tools.py(9 原生语料工具+delegate=10 工具)· .sciscope/skills · mcp_server/mcp_client(MCP双向)· specialists(专员子agent)
  backend/app/api/          5. /api/agent/stream(SSE·session/retry/meta) + search/chat/trends/...
  tui/main.go               6. Go·Bubble Tea 终端客户端（消费 SSE）
  PostgreSQL + pgvector     运行时：papers·chunk_embeddings·paper_embeddings·...
```
> Python 承载全部智能（数据→RAG→agent）；Go 仅作终端客户端，经 SSE 消费、可替换。Agent 由单一 LangGraph StateGraph 编排，`session_id` 映射 LangGraph `thread_id`，运行时用 checkpoint 保存会话态；`retry=true` 会在同一会话线程内触发错误恢复；SSE `meta.phase` 供 TUI 渲染科研工作流阶段条与分组 `/timeline`。详见《项目报告书》§2.7。

## 三、数据底座

- 6 个公开来源（arXiv / PubMed / PMC / OpenAlex / Crossref / DOAJ），形成 **169,324 条原始记录**、**168,447 条分析语料**；其中 **159,382 条**位于 2022–2026 主分析窗口。
- 文件资产口径：`data/processed/paper_chunks.jsonl` 含 **367,861** 行片段；运行时服务库当前以 PostgreSQL 实测为准。文件资产与运行库表用途不同，答辩时需分开表述。
- 覆盖：有摘要 **83.22%**（140,183 篇）、有正文 **10.50%**（17,682 篇）；RAG 片段文件中含 **48,572 个全文片段**、覆盖 **13,400 篇**。
- 原始分区 JSONL：`data/raw_canonical/`；治理后语料：`data/processed/`；分析资产：`data/analysis/`。

## 四、从零一键复现

```bash
# 0. 依赖
make install                       # 后端/Python 依赖
createdb sciscope                  # 本地 PostgreSQL（需先装 pgvector 扩展）
make postgres-schema POSTGRES_DSN=postgresql://<user>@localhost:5432/sciscope
psql sciscope -f infra/postgres/pgvector.sql

# 1. 数据入库 + 模型文件（语料已在 data/processed/）
export SCISCOPE_EMBEDDER_PATH=models/embedder_local/multilingual-e5-base
make full-rebuild                  # 入库→嵌入→趋势/推荐/图谱→数据报告

# 2. 启动本地大模型（另开终端，需先放好 models/llm_local 模型）
make llm                           # OpenAI 兼容服务 :8001

# 3. 启动后端 + 终端智能体
SCISCOPE_DB_DSN=postgresql://<user>@localhost:5432/sciscope make dev
#   Go 客户端需后端 :8000 + 可选 LLM :8001：make tui

# 4. 验证
curl '127.0.0.1:8000/api/search?q=graph+neural+network'
curl -XPOST 127.0.0.1:8000/api/chat -d '{"question":"扩散模型在图像生成中的应用?"}'
make test-backend                  # 后端自动化测试（当前实测 141 passed）
make agent-smoke                   # live agent 黑盒验收（skills/工具预算/库表规模）
```

> 详细步骤、环境变量与故障排查见 `docs/operations/runbook.md`。

## 五、文档口径冲突与实现边界（交接提醒）

以下为本轮交接必须明确的口径差异：

1. `docs/architecture/2026-06-16-sciscope-product-architecture.md` 的“机会生成器 / 交互式 Web 工作台”属于历史路线图目标；当前提交链路为 FastAPI + Go TUI + 单一 LangGraph StateGraph + `.sciscope/skills` 专业工作流 + 9 个原生语料工具 + 1 个 `delegate` 专员工具。MCP 双向互通与三类专员子智能体已落地，但不等同于无限制多代理系统。
2. `README` 与产品文案中的理想化表述不应替代 `Makefile` 与代码作为交付边界；运行行为请以：
   - `/api/agent/stream`
   - `make` 目标链路
   - `backend/app` 实现
   为准。
   其中 `/api/agent/stream` 请求支持 `question/history/session_id/retry`，SSE `meta` 返回 `runtime/node/phase/elapsed_ms/session_id/retry`，用于 TUI 科研工作流阶段条、分组 `/timeline`、会话恢复与 `/retry`。
3. 统计口径统一按 “原始采集 / 清洗语料 / 入库语料 / 报告资产” 四层区分，避免在答辩中混同总量口径。

## 八、技术与合规要点

- **本地优先**：数据、向量索引、嵌入器、生成模型全部本地部署，生成层走 OpenAI 兼容接口可平滑替换（支持涉密/内网场景）。
- **证据可追溯**：问答回答均带 `[n]` 引用，附答案-证据接地度校验（高/中/低置信）。
- **数据来源合规**：6 来源均为公开学术数据。
- **可复现**：每层产物均由 `make` 目标生成，知识产权清晰。
