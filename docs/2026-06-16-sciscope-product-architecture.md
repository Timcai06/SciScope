# SciScope 科研文献洞察智能体产品与技术架构

## 1. 项目定位

### 1.1 产品一句话

SciScope 是一个面向科研人员的文献洞察操作系统，基于 DeepSeek、RAG、知识图谱、多代理协作和时序分析，从海量科技论文中自动发现研究热点、知识脉络、合作网络与潜在选题机会。

### 1.2 核心差异

传统文献系统解决“查找论文”的问题，SciScope 解决“理解领域”和“发现机会”的问题。

系统不以演示 demo 为目标，而以真实科研工作台为目标：

- 让科研人员快速理解一个领域的结构。
- 让科研人员看到主题如何演化、作者如何合作、方法如何迁移。
- 让科研人员获得有证据支撑的问答、推荐、趋势判断和选题建议。
- 让评委看到一个可运行、可扩展、可解释、可复现的产品级智能体系统。

### 1.3 面向评分标准的总策略

- 社会效益：降低科研信息获取门槛，提升青年科研人员与跨学科团队的文献调研效率。
- 经济效益：服务高校、科研院所、企业研发部门、技术情报团队和产业研究部门。
- 技术创新：DeepSeek Agentic Workflow、GraphRAG、动态主题建模、作者合作网络、趋势预测、研究机会评分。
- 格式规范：交付正式项目报告、数据分析报告、系统运行说明、代码工程、演示材料和可复现结果。

## 2. 产品信息架构

### 2.1 整体产品形态

产品采用科研情报工作台布局，而不是传统聊天页面。

主界面分为四个区域：

- 左侧：Research Navigator，研究导航和模块入口。
- 中间：Insight Canvas，主分析画布。
- 右侧：Evidence Panel，证据、论文、图谱路径和分析解释。
- 底部：Temporal Rail，时间轴、主题演化和趋势轨道。

### 2.2 一级模块

1. Research Radar 趋势雷达
   - 输入领域、关键词或研究问题。
   - 输出年度热度、主题分支、关键词演化、未来趋势和机会分数。

2. Evidence Chat 证据型科研问答
   - 中文自然语言提问。
   - DeepSeek 调用检索、图谱、统计和趋势工具。
   - 回答必须带论文证据、图谱关系、时间趋势或统计依据。

3. Knowledge Galaxy 知识星图
   - 可视化论文、作者、关键词、主题和方法之间的关系。
   - 支持时间轴拖动，展示研究主题的出现、融合、分裂和衰退。

4. Collaboration Map 作者合作网络
   - 展示核心作者、合作社区、桥接作者和潜在合作对象。
   - 支持按领域、关键词、年份筛选。

5. Paper Recommender 论文推荐
   - 推荐入门必读、最新趋势、高相关、跨领域启发论文。
   - 每篇论文提供推荐理由和证据来源。

6. Opportunity Generator 研究机会生成器
   - 根据主题热度、增长率、跨领域连接、合作密度和研究空白生成潜力选题。
   - 输出选题名称、研究背景、创新点、代表论文、技术路线、风险和评分。

7. Report Studio 报告生成中心
   - 一键生成数据分析报告、领域综述、选题报告和答辩图表。
   - 支持 Markdown、PDF、DOCX 导出。

## 3. 前端体验设计

### 3.1 视觉方向

前端应呈现“科研情报中控台”的高级感：

- 深色底色，但避免单一蓝黑色，加入低饱和青绿、石墨灰、银白、信号红作为信息层级。
- 高密度信息排布，但保持清晰层级。
- 数据图表和图谱成为主视觉，而不是装饰。
- 所有 AI 输出都配套证据卡片，形成可信的产品气质。

### 3.2 首页即工作台

首页不做欢迎页，进入即显示当前数据集全局洞察：

- 总论文数、年份范围、领域分布。
- 发文量时间趋势。
- 上升最快关键词。
- 核心作者社区。
- 热门主题簇。
- 系统主动生成的今日洞察。

### 3.3 关键交互

- 用户选择一个主题，主画布切换为该主题的趋势雷达。
- 点击趋势峰值，右侧展示对应年份的代表论文和关键词。
- 点击作者节点，展开合作社区和相关研究主题。
- 点击 AI 回答中的证据，主画布定位到对应论文、图谱节点或趋势图。
- 用户可将任意洞察加入报告草稿区，最终生成完整分析报告。

### 3.4 前端技术栈

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- ECharts
- D3.js
- Cytoscape.js
- Framer Motion
- TanStack Query
- Zustand

不建议使用 Streamlit 作为最终产品前端。Streamlit 可以用于算法验证，但最终展示应采用 Next.js 产品级界面。

## 4. 后端与数据架构

### 4.1 后端技术栈

- Python
- FastAPI
- Pydantic
- Pandas
- NumPy
- Scikit-learn
- NetworkX
- BERTopic
- LangGraph

### 4.2 存储设计

- PostgreSQL：存储论文元数据、作者、关键词、主题、分析结果。
- pgvector 或 FAISS：存储论文、摘要、关键词、全文片段的向量索引。
- Neo4j：存储知识图谱和复杂关系查询。
- Redis：缓存问答会话、图表结果和高频分析。
- DuckDB：本地快速分析和离线交付备选。

### 4.3 数据处理流程

1. 数据读取
   - 支持 JSON 和 CSV。
   - 读取论文 ID、标题、摘要、作者、年份、关键词、部分全文。

2. 数据清洗
   - 去重。
   - 空字段处理。
   - 年份规范化。
   - 作者名规范化。
   - 关键词大小写、同义词和中英文术语归并。

3. 文献画像生成
   - 领域分类。
   - 关键词标准化。
   - 摘要向量化。
   - 主题归属。
   - 作者关系抽取。

4. 索引构建
   - 建立向量索引。
   - 建立关键词倒排索引。
   - 建立作者合作图。
   - 建立论文-作者-关键词-主题知识图谱。

5. 分析缓存
   - 年度趋势。
   - 关键词热度。
   - 主题聚类。
   - 作者中心性。
   - 机会分数。

## 5. DeepSeek 智能体架构

### 5.1 DeepSeek 的职责

DeepSeek 不直接“读完整数据库”，而是作为智能体推理中枢：

- 理解用户问题。
- 制定分析计划。
- 调用检索、图谱、趋势和推荐工具。
- 综合结构化证据。
- 生成清晰、可解释、有证据的科研回答。
- 生成研究选题、报告段落和总结。

### 5.2 推荐模型组合

- DeepSeek-V3：常规问答、报告生成、摘要生成、语言润色。
- DeepSeek-R1：复杂推理、研究机会生成、趋势解释、冲突观点分析。

### 5.3 Agentic Workflow

一次复杂查询的工作流：

1. Planner Agent
   - 判断用户意图：问答、推荐、趋势、图谱、报告或选题。
   - 拆分任务。

2. Retriever Agent
   - 从向量库检索相关论文。
   - 使用 reranker 重新排序。

3. Graph Agent
   - 查询论文、作者、关键词、主题之间的关系路径。
   - 返回图谱证据。

4. Trend Agent
   - 计算关键词和主题的时间变化。
   - 返回趋势曲线、增长率和异常点。

5. Recommendation Agent
   - 结合语义相似、主题相关、时间新颖度和图谱距离推荐论文。

6. Writer Agent
   - 生成面向科研人员的结构化回答。

7. Critic Agent
   - 检查回答是否存在无证据判断。
   - 检查是否引用了具体论文、年份、关键词或统计结果。

### 5.4 回答规范

AI 输出必须包含：

- 结论。
- 证据。
- 代表论文。
- 趋势或图谱依据。
- 不确定性说明。
- 下一步建议。

禁止只输出泛泛的自然语言判断。

## 6. 核心算法设计

### 6.1 RAG 与 GraphRAG

基础 RAG：

- 将标题、摘要、关键词、全文片段分别向量化。
- 用户问题向量检索相关论文。
- reranker 提升召回质量。
- DeepSeek 基于检索结果回答。

GraphRAG：

- 先识别问题中的实体和主题。
- 在图谱中查找相关作者、关键词、论文和主题路径。
- 将图谱关系与文本证据一起送入 DeepSeek。
- 回答不仅说明“相关论文有哪些”，还说明“这些论文、作者、关键词之间如何关联”。

### 6.2 动态主题建模

- 使用 BERTopic 对摘要和关键词做主题聚类。
- 按年份切片，计算主题热度变化。
- 识别主题的出现、增长、融合、分裂和衰退。
- 输出主题演化图和主题生命周期。

### 6.3 作者合作网络

- 使用 NetworkX 构建作者合作图。
- 节点为作者，边为共同发表论文。
- 指标包括：
  - Degree Centrality
  - Betweenness Centrality
  - PageRank
  - 社区发现
  - 桥接作者识别

### 6.4 热点趋势预测

- 对关键词和主题构造年度热度序列。
- 使用增长率、移动平均、突现检测和简单预测模型。
- 输出：
  - 上升热点。
  - 衰退热点。
  - 新兴交叉方向。
  - 未来潜力主题。

### 6.5 研究机会评分

每个候选研究方向计算 Opportunity Score：

- 热度增长率。
- 近年论文数量。
- 关键词突现强度。
- 跨领域连接强度。
- 作者合作密度。
- 研究空白程度。
- 代表论文质量。

最终输出：

- 高热成熟方向。
- 高增长新兴方向。
- 低竞争蓝海方向。
- 跨学科迁移方向。

## 7. 系统 API 设计

### 7.1 主要后端服务

- `POST /api/ingest`
  - 导入文献数据。

- `POST /api/index/build`
  - 构建向量索引和图谱索引。

- `POST /api/chat`
  - DeepSeek 证据型科研问答。

- `GET /api/dashboard/overview`
  - 全局数据集洞察。

- `GET /api/trends`
  - 关键词和主题趋势。

- `GET /api/graph`
  - 知识图谱子图查询。

- `GET /api/authors/network`
  - 作者合作网络。

- `POST /api/recommend`
  - 论文推荐。

- `POST /api/opportunities`
  - 研究机会生成。

- `POST /api/reports/generate`
  - 生成报告。

### 7.2 返回数据原则

所有接口尽量返回结构化 JSON，前端负责可视化。

AI 生成内容和数据证据分离：

- `answer`：自然语言回答。
- `evidence`：论文、关键词、图谱路径、统计指标。
- `visualization`：前端可直接绘图的数据。
- `confidence`：置信说明。

## 10. 风险与取舍

### 10.1 数据质量风险

风险：

- 作者同名。
- 关键词不规范。
- 部分全文缺失。
- 领域标签可能不足。

策略：

- 使用规则 + embedding 聚类做关键词归一。
- 作者消歧先采用轻量规则。
- 不依赖全文作为唯一证据，摘要和关键词作为主数据源。

### 10.2 模型幻觉风险

风险：

- DeepSeek 可能生成无证据判断。

策略：

- 所有回答必须绑定 evidence。
- Critic Agent 检查证据缺失。
- 前端展示证据来源。
- 报告中区分数据结论和模型推断。

### 10.3 工程复杂度风险

风险：

- 同时实现图谱、RAG、趋势、前端会导致工程量过大。

策略：

- 先做可运行的数据底座。
- 再做核心智能体。
- 前端先保证主体验完整，再逐步强化视觉。
- Neo4j 可以作为增强模块，若时间紧张则先用 NetworkX + JSON 图数据支撑图谱展示。

## 12. 最终答辩叙事

答辩核心表达：

SciScope 不是一个文献搜索系统，而是一个科研洞察智能体。它通过 DeepSeek 进行复杂推理，通过 RAG 获取文本证据，通过知识图谱组织科研关系，通过时序分析判断热点变化，最终把分散论文转化为可理解、可验证、可行动的科研洞察。

最有冲击力的一句话：

过去，科研人员在文献海洋中寻找信息；现在，SciScope 为他们绘制一张可交互、可追溯、可预测的科研星图。
