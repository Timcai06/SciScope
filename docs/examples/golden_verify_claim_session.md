# SciScope 黄金会话样例：跨语言论断核查

本样例用于赛题交付、产品演示和后续 landing page 展示。它展示 SciScope TUI 中一轮完整科研智能体工作流：用户提出中文科研论断，agent 规划检索路径，调用证据工具，完成自检修正，并输出可复核结论。

## 用户问题

RAG 能否降低科研问答系统中的幻觉风险？请给出证据和判断边界。

## 科研工作流时间线

### 制定研究计划

- 执行计划: 将中文论断改写为英文检索表达 / 检索 RAG 与 hallucination 相关论文 / 用 verify_claim 做跨语言接地核查 / 汇总证据强度和适用边界

### 证据检索

- 论断核查: 检索增强生成能够降低大语言模型回答中的幻觉风险
- 论断核查 · 强支持 · 0.846
- 检索文献: retrieval augmented generation hallucination scientific question answering
- 证据卡 5 篇

### 自检修正

- 自检修正: 证据支持的是“降低无依据回答风险”，不是“完全消除幻觉”；最终结论需保留条件边界。

### 综合回答

- 回答完成

## 研究结论

结论：RAG 通常能够降低科研问答系统中的幻觉风险，尤其是在回答必须绑定论文、数据集、指标或出处时。它的核心价值不是让模型“天然更懂”，而是把回答约束到可检索证据上，使用户可以追溯来源、检查相似度和判断支持等级。

## 依据

- 证据接地：`verify_claim` 返回“强支持”，最高接地相似度约为 `0.846`。
- 机制解释：RAG 将问题先映射到文献片段，再由模型基于片段综合回答，因此能减少凭空生成。
- 边界条件：如果检索库缺失、召回质量差、证据片段与问题不匹配，RAG 仍可能生成错误解释。

## 判断边界

RAG 更准确的表述是“降低幻觉风险并提升可核验性”，而不是“消除幻觉”。因此 SciScope 在 TUI 中把工具调用、证据卡、时间线和 `/timeline` 导出作为一等能力，让用户看到回答是如何被证据约束出来的。

## 可复现入口

- 离线演示：`make tui-demo`
- 环境检查：`sciscope-tui doctor`
- 会话导出：`/export` 或 `sciscope-tui export --last`
- 过程回看：`/timeline`
