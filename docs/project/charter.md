# SciScope 项目方针（Charter）

> 这是 SciScope 的"魂"。它不描述某个功能，而是约束**每一次取舍**。
> 当代码、文档、路线图与本方针冲突时，以本方针为准并回来修它们。
>
> 运行行为的交付边界仍以 `backend/app`、`Makefile`、`/api/agent/stream` 为准；
> 本文件回答的是"为什么"和"往哪走"，不是"现在实现了什么"。

## 一句话定位

**SciScope 不是一个"查文献的软件"，而是一个证据可追溯的「科学证据层」——**
**人可以信任、可以审计、可以推翻，机器（其他 agent）可以调用。**

## 三条不可动摇的方针

### 1. 证据接地是魂，不是脚注

每一句结论都能点回第一手证据（`[n]` 引用 + 接地度校验）。这是 SciScope 唯一
真正抄不走的护城河——大模型时代最稀缺的不是"会答"，是**可信、可审计、可被推翻**。

判定标准（每次改动自检）：

- 结论有没有可点击回溯的出处？
- 系统敢不敢说"证据不足 / 文献里有人反对"，而不是自信地编？
- 人类能不能反驳它、并让它认错？

> 任何让 SciScope 变成"自信黑箱"的改动，无论多强，都违背方针。

### 2. 协议优先，界面可换

产品的本体是「活的知识内核 + 证据图谱 + 协议」，不是任何一个界面。
界面（TUI / IDE 插件 / 浏览器旁路 / 语音）是可替换的皮肤，会不断死去；
协议（SSE / MCP / API）与证据层只需要一个，且赢家通吃。

- 智能只活在 Python 内核（`backend/`、`src/`）；客户端只消费协议，不承载推理。
- 现有 Go TUI 是"协议消费者"，这个决定要守住，不能把智能焊死进界面。

### 3. 成为所有科研 agent 的证据后端

不与 Claude Code / Cursor 争界面，而是**成为它们背后那个给出可追溯科学证据的底座**。
`backend/app/mcp_server.py` 已把工具经 MCP 暴露——这不是附属功能，是**正门**。

> 终极位置：让别的 agent 来调用 SciScope，而不是 SciScope 去模仿它们。

## 北极星：从"回答"到"活的知识体"

当前证据接地（`verify_claim`）用**语义相似度**判"支持"——它量的是"相关"，
不是"蕴含/矛盾"：一句话和它的**反面**会检索到同一批论文、都判"强支持"。
这是护城河上唯一但致命的裂缝。

**补这道裂缝的动作（stance / 蕴含判定），恰好就是"活的知识体"的心脏——**
能判"这篇论文支持还是反驳这句话"的原语，同时修好了核查、又是矛盾检测器。
一个动作，点亮两条愿景。

因此北极星 = **把 `[n]` 从"相关"升级为"蕴含/矛盾"，并把检测到的矛盾沉淀为
一张会随语料生长的"科学争议地图"。** 落地路线见 [roadmap.md](roadmap.md)。

## 数据方针

语料已建齐并评测（16 万篇 / 367K chunks / 1.3 万篇全文），对北极星**量早已够**。

> 停止扩语料。北极星需要的"新数据"不是采集来的，是从现有语料**派生**出来的
> （`claim↔evidence` 立场标注、矛盾表）。力气从"抓更多论文"转向"算出更深结构"。

## 判断一个想法该不该做（三问）

1. 它是否强化"证据可追溯 / 可证伪"这条护城河？（削弱的一票否决）
2. 它是活在**内核+协议**里，还是绑死在某个界面上？
3. 它让 SciScope 更像"被别人调用的证据层"，还是更像"又一个通用助手"？
