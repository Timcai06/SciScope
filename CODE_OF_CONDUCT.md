# Code of Conduct

SciScope is an open-source research-agent project. We want collaboration around
research tooling, data pipelines, agent interfaces, and documentation to be
welcoming, focused, and constructive.

## Our Standards

Examples of behavior that helps the project:

- Be respectful of different backgrounds and experience levels.
- Give specific, actionable feedback.
- Explain tradeoffs when disagreeing.
- Keep discussions focused on the issue, pull request, or design decision.
- Assume good intent while still being precise about technical concerns.

Examples of unacceptable behavior:

- Harassment, threats, insults, or discriminatory language.
- Publicly sharing private information or secrets.
- Dismissing contributors personally instead of addressing the work.
- Spam, off-topic promotion, or repeated disruption.

## Scope

This code of conduct applies to project spaces, including GitHub issues, pull
requests, discussions, documentation, examples, and release channels.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block
participants when behavior harms the project or its contributors. Reports can be
sent privately to the maintainer.

## Maintainer Commitment

Maintainers should apply these standards fairly and should separate technical
review from personal judgment. The goal is a project where serious engineering
discussion can happen without making people feel unwelcome.
